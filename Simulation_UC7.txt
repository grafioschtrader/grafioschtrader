The UC7 is probably the rule-based system. Its purpose is to switch to the simulation environment. What is transferred to the simulation environment with new tenants: the portfolios, securities accounts, and bank accounts. In addition, the strategy intended for the simulation. This strategy should be shared. This means that changes in the original tenant and in the tenant of the simulator are the same. The other entities, such as bank accounts, are linked to the simulation tenant. Of course, these non-shared data can be changed. Switching between the original tenant and the simulation tenant is done via the navigation tree. The global parameters specify the maximum number of simulation environments a tenant can have; the default is 5. When starting the simulation for the first time, existing transactions can also be included, but this is only possible if the previously created strategy is based on UC6. The transactions are transferred up to the date already specified in UC6. AlgoTop will probably need to be extended with this date; so far, it is not persistent. If the transactions are not transferred, the user is presented with a table of all bank accounts in which they can enter the basic balance of these bank accounts. Editing should take place directly in the table. Of course, it could also be entered in the simulation environment, but it is faster to use the start dialog for the first simulation. An icon should be displayed in the navigation tree if a simulation environment already exists for a strategy (AlgoTop). First, update our document @algoimplementationplan.md and @algo.md. Then, the dialog for creating the simulation environment and the simulation environment itself should be implemented.
