===============================================================================
 geoRestrictions — Specification
===============================================================================

DB column: generic_connector_def.geo_restrictions  VARCHAR(512)
Also used by hardcoded connectors via BaseFeedConnector.parseGeoRestrictions().

A single space-separated string that encodes WHERE a connector can fetch data.
When null/empty the connector has no geographic restriction (works globally).

-------------------------------------------------------------------------------
1. Token types
-------------------------------------------------------------------------------

  Token format        Example       Meaning
  ──────────────────  ────────────  ──────────────────────────────────────────
  <COUNTRY_CODE>      US            2-letter ISO 3166 country code.
                                    Connector supports ALL exchanges in that
                                    country, for all asset class categories.

  <MIC>               XNYS          4-letter ISO 10383 MIC code.
                                    Connector supports that specific exchange,
                                    for all asset class categories.

  <MIC>:-<CATEGORY>   XBBC:-ETF     Category exclusion on a specific exchange.
                                    The MIC is supported BUT the named
                                    AssetclassCategory is excluded from it.

  <CC>:-<CATEGORY>    US:-ETF       Category exclusion on a country.
                                    The country is supported BUT the named
                                    AssetclassCategory is excluded from it.

  <CATEGORY> must be one of the AssetclassCategory enum names:
    CURRENCY_PAIR, CRYPTOCURRENCY, NON_INVESTABLE_INDICES, EQUITIES,
    FIXED_INCOME, ETF, MUTUAL_FUND, REAL_ESTATE_FUND, ISSUER_RISK_PRODUCT

-------------------------------------------------------------------------------
2. Rules
-------------------------------------------------------------------------------

  R1  An exclusion token (e.g. XBBC:-ETF) is only meaningful if the
      corresponding geo code (XBBC) is also present as an inclusion — either
      explicitly or implied by its country code.

  R2  A country code includes ALL exchanges of that country. If XNYS is
      also listed alongside US, it is redundant but harmless.

  R3  Exclusion tokens do NOT imply inclusion. "XBBC:-ETF" alone without
      "XBBC" means XBBC is not included at all.

  R4  Multiple exclusions on the same geo are listed as separate tokens:
      "XBBC XBBC:-ETF XBBC:-FIXED_INCOME"

  R5  When geoRestrictions is null/empty AND supportedAssetclassCategories
      is also empty, the connector accepts everything (legacy behavior).

-------------------------------------------------------------------------------
3. Example configurations
-------------------------------------------------------------------------------

  geoRestrictions value             Interpretation
  ────────────────────────────────  ──────────────────────────────────────────
  (null / empty)                    No geographic restriction. Connector
                                    works globally.

  US                                All US exchanges, all categories.

  XNYS                              Only NYSE, all categories.

  US XETR                           All US exchanges + Frankfurt (XETR).

  US DE                             All exchanges in US and Germany.

  XSWX                              Only SIX Swiss Exchange, all categories.

  XETR XFRA                         Frankfurt (XETR) + Frankfurt floor (XFRA).

  XBBC XBBC:-FIXED_INCOME           London (XBBC) but NOT fixed income.

  US US:-CRYPTOCURRENCY              All US exchanges but NOT crypto.

  XETR XETR:-ETF XETR:-MUTUAL_FUND  Frankfurt but NOT ETFs or mutual funds.

  US XETR XETR:-FIXED_INCOME        All US + Frankfurt without fixed income.

-------------------------------------------------------------------------------
4. Frontend mapping
-------------------------------------------------------------------------------

  The frontend splits geoRestrictions into two form fields for editing:

  ┌─────────────────┬──────────────┬────────────────────────────────────────┐
  │ Form field       │ Widget       │ Content                                │
  ├─────────────────┼──────────────┼────────────────────────────────────────┤
  │ geoInclusions    │ TreeSelect   │ Country codes and MIC codes.           │
  │                  │ (checkbox,   │ 2-level tree: Country → MIC.           │
  │                  │  no prop.)   │ No propagation: checking a country     │
  │                  │              │ does NOT auto-check its MIC children.  │
  │                  │              │ Each level is independent.             │
  │                  │              │                                        │
  │                  │              │ Tree source: configured Stockexchange  │
  │                  │              │ entities (not all ISO 10383 MICs).     │
  ├─────────────────┼──────────────┼────────────────────────────────────────┤
  │ geoExclusions    │ TreeSelect   │ Geo → Category exclusion tree.         │
  │                  │ (checkbox,   │ 2-level tree: GeoCode → Category.      │
  │                  │  WITH prop.) │ WITH propagation: checking a geo code  │
  │                  │              │ checks all its category children.      │
  │                  │              │                                        │
  │                  │              │ Tree is rebuilt dynamically whenever    │
  │                  │              │ geoInclusions changes. Only geo codes  │
  │                  │              │ currently selected in geoInclusions     │
  │                  │              │ appear as parent nodes. Each parent has │
  │                  │              │ AssetclassCategory children (translated │
  │                  │              │ labels).                               │
  │                  │              │                                        │
  │                  │              │ Semantics: checking a category means   │
  │                  │              │ "EXCLUDE this category for this geo".  │
  │                  │              │ An unchecked (empty) tree means no     │
  │                  │              │ exclusions.                            │
  └─────────────────┴──────────────┴────────────────────────────────────────┘

  On SAVE:
    1. Collect checked nodes from geoInclusions → geo codes
    2. Collect checked leaves from geoExclusions → GEO:-CATEGORY tokens
    3. Merge into single geoRestrictions string

  On LOAD:
    1. Split geoRestrictions into tokens
    2. Plain tokens (no ":-") → check matching nodes in geoInclusions TreeSelect
    3. Build geoExclusions tree from those inclusions
    4. Exclusion tokens (":-") → check matching GEO:CATEGORY nodes in
       geoExclusions TreeSelect. The geo code before ":-" is also added
       as an inclusion (rule R1).

  TreeSelect selection format (PrimeNG checkbox mode):
    { [nodeKey: string]: { checked: boolean, partialChecked: boolean } }

  geoInclusions:  propagation disabled, partialChecked never set by widget.
  geoExclusions:  propagation enabled, parent shows partialChecked when some
                  (but not all) category children are checked.

-------------------------------------------------------------------------------
5. Backend evaluation (BaseFeedConnector.supports)
-------------------------------------------------------------------------------

  Input: mic, countryCode, assetclassType, specialInvestmentInstruments

  Step 1 — If supportedAssetclassCategories is empty → return true (no filter)

  Step 2 — Geo check: if supportedGeoRestrictions is non-empty, at least one
           of mic or countryCode must be in the set. Otherwise → return false.

  Step 3 — Category check: find an AssetclassCategory that matches the
           assetclassType + specialInvestmentInstruments combination.
           If none matches → return false.

  Step 4 — Exclusion check: if geoExclusions contains the matched category
           for the given mic or countryCode → return false.

  Step 5 → return true.
